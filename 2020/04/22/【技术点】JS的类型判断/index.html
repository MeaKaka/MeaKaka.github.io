【技术点】JS的类型判断

Posted on 2020-04-22 | Post modified: 2020-07-16
Words count in article: | Reading time ≈

JS有哪些类型

JS中的类型主要分为两类基本类型和引用类型。
基本类型有以下7种：

  • Number
  • Boolean
  • String
  • Symbol
  • Null
  • Undefined
  • BigInt

而Object属于引用类型

Array本质上是Object

typeof

typeof主要是判断操作数的基本类型,除此之外还支持function类型的判断。

new Boolean() 是JS封装的构造函数，得到是Boolean对象与基本的Boolean类型不同。

typeof new Boolean(false) “object”

由此引申出一个问题对于一个false的数值(任意数值)要连续2次进行new Boolean的转换会得到true。因为new Boolean()会返回一个对象，而new Boolean中的参数为任意对象时都会返回true

同理，new String、new Number也是同样的情况

instance of

instance of的原理是基于原型链的。

只要在原型链上就会返回true。
正是由于是通过原型链进行判断的，所以对于一些基本类型，instanceof就无能为力了

instanceof主要是针对引用类型的判断，只要是引用对象原型链上肯定会有Object，除此之外还可以用来判断是否继承了另一对象。

Object.prototype.toString

toString() 方法返回一个表示该对象的字符串。
其他对象可以通过call/apply的方式去调用Object的toString方法。

Object.prototype.toString的执行规范

When the toString method is called, the following steps are taken:

  1. If the this value is undefined, return “[object Undefined]”.
  2. If the this value is null, return “[object Null]”.
  3. Let O be ! ToObject(this value).
  4. Let isArray be ? IsArray(O).
    1. If isArray is true, let builtinTag be “Array”.
  5. Else if O has a [[ParameterMap]] internal slot, let builtinTag be “Arguments”.
  6. Else if O has a [[Call]] internal method, let builtinTag be “Function”.
  7. Else if O has an [[ErrorData]] internal slot, let builtinTag be “Error”.
  8. Else if O has a [[BooleanData]] internal slot, let builtinTag be “Boolean”.
  9. Else if O has a [[NumberData]] internal slot, let builtinTag be “Number”.
  10. Else if O has a [[StringData]] internal slot, let builtinTag be “String”.
  11. Else if O has a [[DateValue]] internal slot, let builtinTag be “Date”.
  12. Else if O has a [[RegExpMatcher]] internal slot, let builtinTag be “RegExp”.
  13. Else, let builtinTag be “Object”.
  14. Let tag be ? Get(O, @@toStringTag).
  15. If Type(tag) is not String, set tag to builtinTag.
    Return the string-concatenation of “[object “, tag, and “]”.

This function is the %ObjProto_toString% intrinsic object.

可以看出在调用toString之后，首先要判断value是否是undefined/null，然后再判断是否是Array、Arguments、Function…等。

可以看出toString能够识别出如下的数据类型。

  • undefined
  • Null
  • Arguments
  • Array
  • Boolean
  • Date
  • Error
  • Function
  • JSON
  • Math
  • Number
  • Object
  • RegExp
  • String

    总结 [对比/差别]

  1. 利用typeof不能区分null和object，也不能区分array和object，不能细分所有引用对象
  2. 使用instanceof的前提是明确对象和要比较的构造函数，由于instanceof是基于原型链的所以针对的数据是引用类型
  3. Object.prototype.toString可以说是全能型选手，可识别的种类较多
  4. Array除了可以使用instanceof、Object.prototype.toString外还可以使用Array.isArray()进行判断。

    isArray()首先要判断参数是否是Object，如果不是就直接返回false，否则判断参数是否是「Array Exotic Objects」/proxy（递归）

关于「Array Exotic Objects」的定义：
是一个特殊的对象，有一个不可编辑的属性length。

An Array object is an exotic object that gives special treatment to array index property keys (see 6.1.7). A property whose property name is an array index is also called an element. Every Array object has a non-configurable “length” property whose value is always a nonnegative integer less than 232. The value of the “length” property is numerically greater than the name of every own property whose name is an array index; whenever an own property of an Array object is created or changed, other properties are adjusted as necessary to maintain this invariant. Specifically, whenever an own property is added whose name is an array index, the value of the “length” property is changed, if necessary, to be one more than the numeric value of that array index; and whenever the value of the “length” property is changed, every own property whose name is an array index whose value is not smaller than the new length is deleted. This constraint applies only to own properties of an Array object and is unaffected by “length” or array index properties that may be inherited from its prototypes.
