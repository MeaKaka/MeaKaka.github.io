时间

定义

时间可以分为

  • UTC（协调世界时）又称世界统一时间、世界标准时间、国际协调时间，指的是time中的世界时间标准。
  • 当地时间指的是指执行JavaScript的客户端电脑所设置的时间。
  • GMT（格林威治标准时间）一般指世界时，与UTC时间戳相一致。
  • CST（中央标准时间）可视为美国、澳大利亚、古巴或中国的标准时间。

表示时间的方式

时间字符串有两种标准分别是RFC-2822 和ISO 8601标准

  • ISO 8601
    • YYYY-MM-DDTHH:mm:ss.sssZ
    • 需要注意的是-连接符才是标准连接符
    • 如果你不提供“时间”（也就是 T 分隔后的内容），得到的其实是 UTC 时间
  • RFC-2822
    • Wed Mar 25 2015 09:56:24 GMT+0100
  1. 由于浏览器之间的差异与不一致性，强烈不推荐使用Date构造函数来解析日期字符串或使用与其等价的Date.parse。
  2. 对 RFC 2822 格式的日期都可以支持。
  3. 但是在 ISO 8601 格式中，仅有日期的串 (例如 “1970-01-01”) 会被处理为 UTC，其他字符串会被处理为本地时间。

Date

创建一个 JavaScript Date 实例，该实例呈现时间中的某个时刻。Date 对象则基于 Unix Time Stamp，即自1970年1月1日（UTC）起经过的毫秒数。

  1. Date.prototype.getDate()根据本地时间返回指定日期对象的月份中的第几天（1-31）
  2. Date.prototype.getUTCDate()根据世界时返回特定日期对象一个月的第几天（1-31）

new Date() 传入参数

当Date作为构造函数调用并传入多个参数时，所定义参数代表的是当地时间，参数可以为时间戳、或符合格式的字符串。

1. 在ios或safari中new Date()的中使用ISO样式的字符串一定要符合格式
2. 如果字符串中含有时分秒。chrome可以解析「/」和「-」连接符，Safari不能解析「-」连接符
3. 如果将「-」连接的时间字符串中添加一个「T」则safari可以解析，此时「/」+「T」为无效时间字符串形式。在加上T之后，safari将出入参数默认为格林威治时间，但是chrome还是按照本地时间解析。（加上Z后则表示格林威治时间）
4. 如果只有年月日不含时间则ISO标准默认将参数看作格林威治时间，「-」标准连接符的时间比「/」连接符的时间多八个小时。

Date.parse()

支持 RFC-2822 和ISO 8601 的标准，将字符串转换为时间戳

如果字符串能正确解析就返回自 UTC 时间 1970-01-01 00:00:00 经过的毫秒数，否则返回 NaN。

总结

考虑到浏览器的兼容性，在使用时间时最好选择RFC-2822格式或「/」连接符。

Antd的DatePicker的onChange函数中有两个参数，分别是value和dateString，value里存放的是含有RFC-2822格式的时间的对象，dateString是用-连接无T时间字符串。
