【技术点】从浏览器高度说开去

Posted on 2020-03-30 | Post modified: 2020-03-31
Words count in article: | Reading time ≈

场景概述

在做数据加载的时候进行需要考虑到「无限加载」的情况，特别是移动端，由于屏幕尺寸不适合利用pagination做分页控制。这时候常见的解决办法是监听用户的页面滚动。

当滚动到屏幕底部时触发下一页的请求，同时将请求得到的数据拼接到原有数据上。这也是长列表性能优化的一种方式。

针对select组件上的optionList的加载也可以利用这种方式，缺点在于搜索需要交给后端去实现

上述问题的关键就转变为了「如何获取屏幕高度」，判断列表滚动到屏幕底部。

知识点

document对象

document对象获取的是文档信息

网页视图
  • document.documentElement.clientHeight：元素可见区域的高度（包括内边距，但不包括垂直滚动条、边框和外边距）（height + padding）
  • document.documentElement.offsetHeight：元素的像素高度（包括内边距、滚动条、边框和外边距）（height + padding + border）
  • document.documentElement.offsetTop：返回当前元素相对于其 offsetParent 元素的顶部内边距的距离。
  • document.documentElement.scrollHeight：元素总内容的高度（包含元素的内边距，但不包括边框，外边距或垂直滚动条）
  • document.documentElement.scrollTop：元素内容的滚动高度

    documentElement可以是任意的合法document对象，可以通过事件触发的e.target获取，如果直接写作documentElement则获取到的结果与body相同

    body对象
  • document.body.clientHeight/clientWidth：网页可见区域(body区域)高/宽
  • document.body.offsetHeight/offsetWidth：网页可见区域高/宽(包括边线的宽)
  • document.body.scrollHeight/scrollWidth：网页正文全文高/宽
  • document.body.scrollTop/scrollLeft：网页被卷去的高/宽

    需要注意在chrome上使用document.body.scrollTop会一直等于0，从而得不到滚动的高度。可以通过Math.max(document.body.scrollTop, document.documentElement.scrollTop)实现浏览器的兼容

window对象

window对象是整个浏览器的相关信息

screen对象

screen对象代表屏幕信息

  • window.screenTop/screenLeft： 浏览器上边界/左边界到屏幕的距离（可以判断浏览器是否最大化了）
  • window.screen.height/width：屏幕的高度/宽度，含书签栏
  • window.screen.availHeight/availWidth：屏幕的最大可用高度/宽度，除去导航栏之后的可用高度
    浏览器相关
  • window.innerHeight：获取浏览器视觉视口高度（包括滚动条，对比document.documentElement.clientHeight）。
  • window.outerHeight：获取浏览器窗口外部的高度。表示整个浏览器窗口的高度，包括侧边栏、窗口镶边和调正窗口大小的边框。
  • window.scrollY/scrollX: 文档垂直/水平滚动的像素值
  • window.pageYOffset/pageXOffset: scrollY/scrollX的别名(对比document.documentElement.scrollTop)

如果想获得文档内容的滚动高度可以通过window.scrollY / window.pageYOffset / document.documentElement.scrollTop / document.body.scrollTop 差别在于浏览器的兼容.

为了跨浏览器兼容，用window.pageYOffset 代替 window.scrollY （旧版本的IE（<9）两个属性都不支持）

判断Doctype的类型，如果是标准模式用document.documentElement.scrollTop;如果是混杂模式，用document.body.scrollTop.

实例

通过了解确定高度的不同方法，可以选用「卷曲高度」+「视图高度」= 「总页面高度」的表达方法

拓展

与高度相关的场景还有图片的懒加载，需要判断待加载的图片是否滚动到了可视区域，可以通过getBoundingClientRect()获取到物体的位置。

getBoundingClientRect

element.getBoundingClientRect()返回元素的大小及其相对于视图的位置。


图片来源MDN web doc

  1. top: 元素上边距可见窗口顶部的长度
  2. left: 元素右边距可见窗口左边的长度
  3. right: 元素右边距可见窗口左边的距离
  4. bottom: 元素下边距可见窗口上边的距离
  5. width: 元素宽度（right-left）
  6. height: 元素高度（bottom-top）

可以通过比较top的值和可视窗口本身的高度大小来判断是否元素是否到达可视窗口。
