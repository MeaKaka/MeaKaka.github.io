JS

String

字符串与数字的转化
  • parseInt(string，(基数))
  • parseFloat()
  • 按位非~~string，仅限于整数，
    • 例:~1.2=1；\“1.2”=1；~~”234hello234”=0
    • 原理是按位翻转
  • Number
    • 例: Number(“023”) = 23; Number(023) = 19 (八进制)；Number(‘23k’) = NaN
    • 效率慢几乎不使用
  • 一元运算符
    • number*1，number/1
    • number-0（+0不可以是因为+操作会把0看作是字符串）
    • +string
string的一些方法
  • string.trim()用来删除字符串两边的空白
正则匹配与替换
  • string.match()表示正则匹配
  • string.replace(‘正则’，replaceValue)表示替换
屏幕高度
  • 屏幕
    • screen.height/screen.width 屏幕高度
    • screen.availHeight/screen.availWidth 屏幕可用高度，表示软件最大化时的高度
  • 浏览器（191121/Document/浏览器高度）
    • window.outerHeight/outerWidth 浏览器高度/宽度
    • window.innerHeight/innerWidth 浏览器页面可用，减去边框工具条
  • 页面
    • body.offsetHeight/offsetWidth body的总高度
    • body.clientHeight/clientWidth body展示的总高度

      *动态显示头部导航的方式就是监听滚动条的位置

      *element.getBoundingClientRect用来获取元素距离上下左右边距页面上、左的位置

  • scroll
    • scrollTop 获取/设置内容向上滚动的像素值
    • scrollHeight 一个元素的所含的高度的测定，包括由于溢出内容在屏幕上不可见的

      *利用element.scrollHeight - element.scrollTop === element.clientHeight关系式判断是否滑动到最底部

      *谷歌浏览器不支持document.documentElement.scrollTop，会出现值一直为0的问题，可以利用document.body.scrollTop进行代替

      *可以利用document.compatMode获取当前浏览器采用的渲染方式。值为BackCompat时，浏览器客户区宽度是document.body.clientWidth；值为CSS1Compat时，浏览器客户区宽度是document.documentElement.clientWidth。

CSS

  • border-radius用来实现圆角（bordor-top-left-radius）
  • background-origin用来设置background的范围（content-box、padding-box…）
  • flex布局中实现水平垂直居中
    • justify-content：center（主轴的对齐方式）
    • align-items：center（交叉轴的对齐方式）
  • background-size
    • contain原图
    • cover覆盖背景
    • 默认repeat
  • background-position-x
    • 设置水平方向的位置
    • left/right/center
    • right/left value
  • 图片透明而文字不透明的方式：额外多一层蒙版。
    • 图片本身是没法设置透明度的，可以在图片上设置一层含有一定不透明度的背景色
  • 浮动
    • 浮动的影响
      • 元素溢出
      • 不会换行(对话框的开发过程中)
    • 解决方法
      • <br>
      • clear:both/left/right
  • :host
    • CSS中的伪类函数，shadow DOM
    • web components的封装特性，将html标签结构、css样式隐藏起来，与其他代码分离
    • :host在shadow DOM之外使用时没有任何效果

HTML5

DOM

scrollIntoView
  • Element.scrollIntoView()让当前元素滚动到浏览器窗口的可视区域内
    • 可传参数Boolean
      • true 元素顶端与可视区顶端对齐
      • false 元素底部与可视端底部对齐
    • 可传参数Obj
      • behavior：auto/instant/smooth（瞬间/平滑）
      • block：start/center/end
  • Element.scrollIntoViewIfNeed()当元素不在可是区域时才发生滚动
    • 只有Boolean参数
getElementById()

可以得到对应id的元素的value，也可以利用给value赋值的方式将对应元素的值置为空。
但是需要注意这种置空方式不能用在antd之类的封装组件上，因为实际的id是不同的。

datalist

可以实现搜索和输入结合。但是不能控制位置以及与服务器数据绑定。
datalist 及其选项不会被显示出来，它仅仅是合法的输入值列表。
使用时与input 元素的 list 属性来绑定。

小程序

  • 小程序中的this.setData是针对于page而言的，而非app对象中函数，所以在app对象中设置global对象的值是调用this.setData会出现not function的错误。（可以直接设置app.global.data = data）。

  • not function的错误可能发生也函数嵌套函数中，要注意传递this。

  • 小程序的request请求中的method仅支持get、push、post

  • setTimeout后就执行clearTimeout可能回调函数还没执行，定时器就被清掉了。

  • 飞书小程序实质上不会做域名安全性的校验

  • 小程序官方是自带showModal、showToast方法的，不需要自己写样式

Git

  • 只有owner和maintainer可以直接对master进行push操作，developer没有权限，可以新建branch再merge

网络

  • fetch 默认对服务端通过Set-Cookie头设置的cookie也会忽略，若想选择接受来自服务端的cookie信息，也必须要配置credentials选项
    • credentials 是Request接口的只读属性，用于表示用户代理是否应该在跨域请求的情况下从其他域发送cookies。这与XHR的withCredentials 标志相似，不同的是有三个可选值（后者是两个）：
      • omit: 从不发送cookies.
      • same-origin: 只有当URL与响应脚本同源才发送 cookies、 HTTP Basic authentication 等验证信息.(浏览器默认值,在旧版本浏览器，例如safari 11依旧是omit，safari 12已更改)
      • include: 不论是不是跨域的请求,总是发送请求资源域在本地的 cookies、 HTTP Basic authentication 等验证信息.
