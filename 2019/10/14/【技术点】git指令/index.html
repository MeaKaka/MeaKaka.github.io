Posted on 2019-10-14 | Post modified: 2020-03-17

基础指令

git branch 新建分支

git commit 在当前分支下提交代码

git checkout X 切换分支为X分支，只能切换到指定分支

当发生冲突时，git checkout breach filename能够更新你工作区的文件或者目录从其他的breach，而不用将整个branch merge。

git merge A 将A分支合并到当前分支上

git fetch 将远程主机的最新内容拉到本地

git branch –delete name 删除分支（git branch -d）

git push origin –delete branchName

git rebase X 线性合并X分支和当前分支，将当前分支上提交的commit看作一个补丁，在X分支的后续位置提交所谓的补丁形成一个完成合并；rebase和merge的差别在于分支轨迹是否是线性的
git pull 取回远程主机某个分支的更新，再与本地的指定分支合并，完整格式为

进阶指令

改变位置

HEAD

HEAD 总是指向当前分支上最近一次提交记录。

大多数修改提交树的 Git 命令都是从改变 HEAD 的指向开始的。

如果想看 HEAD 指向，可以通过 cat .git/HEAD查看， 如果 HEAD 指向的是一个引用，还可以用 git symbolic-ref HEAD 查看它的指向。

HEAD一般来说默认指向分支名，分离的 HEAD 就是让其指向了某个具体的提交记录而不是分支名。

相对引用

相对引用，可以从一个易于记忆的地方（比如 bugFix 分支或 HEAD）开始计算。

^表示向上移动一个提交记录，～num表示向上移动num个提交记录

将分支切换到branchName的分支，同时移动一个提交单位

切换分支

git branch的-f 选项让分支指向另一个提交，移动master指向，当进行多次rebase后可以移动master指向head

将 master 分支强制指向 HEAD 的第 3 级父提交

而git checkout的不同参数可以表示移动到不同的父提交上

撤销变更

git reset，和git revert是两种常见的撤销变更的方式。

git reset即撤销，分支记录回退（git reset 对应位置）（在本地分支做处理）

git revert即回滚，撤销更改并分享给别人（新建一个commit来回滚到之前的commit）（针对远程分支做处理）

创建关联

–track 和 –set-upstream-to用于关联本地分支和线上分支,–set-upstream命令已经不支持了

其他指令

git tag

添加标签，可以永远指向某个提交记录的标识。当软件发布新的大版本，或是修正一些重要的 Bug、增加了某些新特性时，可以通过git tag的命令为这些改变后的提交增加一个标识。

存在两种标签类型，分别是轻量级标签和带注释标签。轻量级标签是一个简单的指向一次提交的带名字指针。你随时都可以将它指向另一个提交。

带注释标签是一个指向标签对象的带名字指针，带有自己的信息和历史。因为有自己的信息，它可以根据需要用GPG签名。

git describe

用来描述离你最近的锚点（也就是标签），显示离当前提交最近的tag。如果符合条件的tag指向最新提交则只是显示tag的名字，否则会有相关的后缀来描述该tag之后有多少次提交以及最新的提交commit id。不加任何参数的情况下，git describe 只会列出带有注释的tag。

git bisect

查找产生 Bug 的提交记录的指令

git cherry-pick

git cherry-pick <提交号>…表示将一些提交复制到当前所在的位置（HEAD）下，如将xx分支上的工作复制到 master分支

实际场景应用

分支开发时，master代码有更改怎么办？

——如何将master分支合并到本地分支上

两种方式，一种是开发完当前分支之后将其merge到master分支上；另一种是将修改后的master合并到本地分支上，然后在更新后的当前分支上再进行修改。第一种方法就不赘述了，主要说第二种方法。

第二种方法可以用merge和rebase两种方式来实现。

git merge（pull：git fetch + git merge）从master分支merge 到当前分支，但是这会产生一个新的提交

git rebase（git rebase master）从当前分支和master分支进行线性合并，依据rebase命令的性质会将老的commit丢弃生成新的commit。

撤销还未push的commit

HEAD^表示上一个版本，也可以写成HEAD～1，当要表示向上n个版本时可以用HEAD～n

  • –soft表示不删除工作空间改动代码，撤销commit，不撤销git add .
  • –mixed表示不删除工作空间改动代码，撤销commit，并且撤销git add . 操作，为默认操作
  • –hard表示删除工作空间改动代码，撤销commit，撤销git add .操作，注意完成这个操作后，就恢复到了上一次的commit状态

如果想要修改注释可以通过git commit –amend，修改提交commit的附带信息。

想要撤销已经push的commit可以通过强制覆盖（–force）达到效果，git push origin 分支名 –force。
