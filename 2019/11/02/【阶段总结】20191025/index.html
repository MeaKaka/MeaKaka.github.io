【阶段总结】20191025

Posted on 2019-11-02 | Post modified: 2020-03-09

JS

Array

  • Array.find() 方法返回数组中满足提供的测试函数的第一个元素的值。否则返回 undefined，相当于实现了遍历匹配返回。

  • Array.join()方法，对比string.split()，只有array有join方法，如果array为空的话返回空字符串。利用join+\n实现换行，返回的仍是字符串，除非对\n换行符有识别，html对\n没有识别。replaceAll可以利用split和join来实现。

  • Array.sort()方法会改变原数组，因为原地排序。对比生成新数组的命令，map 、filter。

  • array.concat（）合并两个数组，返回一个新数组

String
  • JS截取字符串，substr(start,[len])、slice(start,[end])、substring(start,[end])，都可以只接收一个参数，用来表示起始位置start。
object
  • Object.defineProperty()方法会直接在一个对象上定义一个新属性，或者修改一个对象的现有属性， 并返回这个对象。
  • 如果某个属性的 configurable 为false, 那么:
    • 将不能删除此属性, 即 delete obj.xxx 无效, 在严格模式下直接报错. 不能修改相关配置信息，属性内函数或者变量（delete返回值是false的情况）
    • writable的可配置性依据configurable
    • 虽然 configurable 是 false, 但只要 writable 是 true, 便可以重新定义 value
JS时间格式化

New Date（时间戳），将时间戳转化为日期格式，常见的命令有getYear()、getMonth()，这里需要注意的getDate()和getDay()的差别。

  • myDate.getDate(); //获取当前日(1-31)
  • myDate.getDay(); //获取当前星期X(0-6,0代表星期天)
  • .toFixed(n); //取n位小数

需要注意的是原生JS中Date里是没有format方法的，所以Date.format(“yyyy-mm-dd”)是会报错的，可以通过getYear()等函数的求值进行拼接后实现format方法。

Date.toString()返回的是一个字符串，表示该Date对象，与期望的格式化输出还是不一样。

需要注意的是安卓/ios、谷歌/safari在针对时间处理上的差异。ios不支持2019-12-24 11:00这样格式的时间,只能针对2019/12/24 11:00或是2019/12/24T11:00的时间格式进行处理。

Date.parse()返回时间戳

new Date()返回字符串式时间格式，+Date()、Date().valueOf()、Date.now()都可以返回对应的时间戳。

以上两个方法在参数为null时可以正常处理，当参数为字符串时需要注意传入时间的格式

moment.js中提供了针对Date的加减操作，moment().add(1,”day”)，在不使用该库的情况下我们可以使用get和set实现时间的加减。

forEach&&map

  • forEach(): 针对每一个元素执行提供的函数(executes a provided function once for each array element)。

  • map(): 创建一个新的数组，其中每一个元素由调用数组中的每一个元素执行提供的函数得来(creates a new array with the results of calling a provided function on every element in the calling array)。

forEach()方法不会返回执行结果，而是undefined。也就是说，forEach()会修改原来的数组。而map()方法会得到一个新的数组并返回。所以利用forEach进行数据的遍历展示<span>会失败

arguments

arguments对象是所有（非箭头）函数中都可用的局部变量，维护所有传递到这个函数中的参数列表。

正则表达

匹配

要求输入的内容前后不能有空格，中间可以有空格

比如要求匹配IP

在匹配ip的时候还需要考虑这种情况，对于无效ip是要截取有效的ip字段还是完全不做考虑，这时候可以利用正则里的正(反)向肯(否)定预查,对捕获文本的前后内容进行指定。

matchAll&&match&&exec
regexp.exec(string)

字符串中如果有匹配值就返回该匹配值，否则返回null。其实返回值是一个含匹配值的数组，数组里有index匹配的位置和input输入的值。

需要注意的是exec永远只返回第一个匹配，即使有g。但是在多次调用的时候如果有g将在上一次匹配返回的地方接着匹配。

在含分组匹配中除了全局的匹配外还会在返回的数组中包含第一个匹配分组的值。

String.match (regexp)

如果使用了g则会返回与完整表达式匹配的所有结果，但不会返回捕获组；

如果未使用g则仅返回第一个完整匹配和与其相关的捕获组。（与regexp.exec相同）

String.matchAll(regexp)

适用于分组捕获，返回符合条件的迭代器，利用[…str.matchAll(regexp)]可以得到最终的数组嵌套数组

网络交互

Unexpected token < in JSON at position 0

这些错误发生在当你向服务器发送请求，返回值不是JSON而用JSON的方法解析的时候，发生这种情况的代码可能是这样的。（很有可能是请求的的参数出错了，导致后端返回的值是500 service error，从而导致无法解析）

provisional headers are shown

请求里面provisional headers are shown(显示临时报头) 出现的情况很多，但原因是多样的。

大部分的原因是请求被拦截，特别是请求跨域被拦截

credentials

credentials 是Request接口的只读属性用于表示用户代理是否应该在跨域请求的情况下从其他域发送cookies。这与XHR的withCredentials 标志相似，不同的是有三个可选值（后者是两个）：

  • omit: 从不发送cookies.
  • same-origin: 只有当URL与响应脚本同源才发送 cookies、 HTTP Basic authentication 等验证信息.(浏览器默认值,在旧版本浏览器，例如safari 11依旧是omit，safari 12已更改)
  • include: 不论是不是跨域的请求,总是发送请求资源域在本地的 cookies、 HTTP Basic authentication 等验证信息.
HTTP2

Http2 头部压缩的实质是用较少的字符表示等量的信息，用静态表和动态表来实现字典（映射）的功能。

  • 静态表，已知键值对如method：get用一个编码去表示，或者是只有key没有value
  • 动态表是利用已发送的请求进行出现过的键值对的记录，允许重复的记录出现，大小有要求

总结来说，静态表可以是所有浏览器通用的，但是动态表是于某个主机绑定的。

——可以直接利用索引值进行后续的匹配，索引空间地址是从1开始的，0的话会被视为错误，因为动态表在不断的更新（但是大小有限制），所以后续的压缩程度会更大。

fetch blocked mixed content

如果您的网站提供HTTPS页面，则默认情况下将阻止通过HTTP在该页面上传递的所有内容，同时浏览器提示fetch blocked mixed content。

React

  • 装饰器（Decorator）是一种与类（class）相关的语法，通过提供一种方便的手段，增加或修改类的功能。因此@connect类不能应用于函数组件。
  • useRef会保存每一次生成的闭包环境，以免在render过程中生成多个函数，起不到防抖的效果。
  • 当一个组件的 props 或 state 变更，React 会将最新返回的元素与之前渲染的元素进行对比，以此决定是否有必要更新真实的 DOM。React的diff算法会比较key是否一样，如果一样就不会重新render了，因此当一个DOM元素已经生成则二次渲染时会根据key进行DOM的修改。
    • defaultvalue表示默认的初始值，会被新值覆盖掉。
    • 对于同一个DOM元素来说，如果有value值了修改defaultValue是没有作用的。
  • 其实react在重新渲染组件的时候，当某个父组件发生变化，并不会渲染所有的子组件，react渲染流程图。
  • 可以通过生命周期中shouldComponentUpdate返回的false和true来判断是否要重新渲染
  • 可以继承React.PureComponent以代替手写shouldComponentUpdate，pureComponent默认使用了浅复制，浅比较可能会有遗漏。当数据来源是一个对象只是里面的值被改变，在渲染时可能会被当作没有变化，所以在处理深层嵌套对象的时候一定要考虑对象的更新方式。

Antd

  • antd的form 的getFieldDecorator里的id是支持嵌入式的，因此才进行拼接的时候要考虑拼接字符串中“.”的存在。
  • Upload上传文件时，后端框架只支持名称里含file的请求
  • Antd form组件，可以使用 this.props.form.setFieldsValue 来动态改变表单值
  • form.getFieldValue(‘变量名’)，得到form对应变量名的值
  • table组件中的defaultExpandedRowKeys字段对异步数据无效，需要用 expandedRowKeys 控制，因为首次渲染的时候数据还没有出现

HTML

  • 在设置<a>标签时，如果target设置为_blank即在另一页面打开，则一定要设置rel=noopener noreferrer，以免新页面能访问旧页面的window对象（在B页面中通过window.reopner即可获取A页面的window对象），同时能设置rederrer，知道页面跳转来源。
  • 几乎所有的html元素都有onclick属性，除了<base>、<bdo>、<br>、<head>、<html>、<iframe>、<meta>、<param>、<script>、<style> 和 ]<title>。
  • <link rel=”preload”>在link标签中可以设置加载方式，使用 preload 可以来加载最重要的资源，比如图像，CSS，JavaScript 和字体文件。
    • 这与浏览器预加载不同，浏览器预加载只预先加载在HTML中声明的资源。
    • preload 与 prefetch 不同的地方就是它专注于当前的页面，并以高优先级加载资源，
    • Prefetch 专注于下一个页面将要加载的资源并以低优先级加载。同时也要注意 preload 并不会阻塞 window 的 onload 事件。
    • preconnect 允许浏览器在一个 HTTP请求正式发给服务器前预先执行一些操作，这包括 DNS 解析，TLS 协商，TCP 握手，这消除了往返延迟并为用户节省了时间。
    • Window onload事件 在文档加载完成后能立即触发，并且能够为该事件注册事件处理函数
    • rel 属性规定当前文档与被链接文档之间的关系

Webpack

webpack的setting要重新编译，而不是热加载。

路由

  • this.props.match.match、this.props.match.location、this.props.match.history如果我们没用react-router中的route组件包裹组件，我们就需要通过上面形式获得url的信息，都是在match里面；而如果我们了react-router的route组件，我们就可以直接通过this.props.match,this.props.location,this.porps.history来获取url信息。
    • location可以通过search字段得到url的query的信息
    • Match可以通过params得到路由参数的值
    • history：包含了组件可以使用的各种路由系统的方法，常用的有 push 和 replace，两者都是跳转页面，但是 replace 不会引起页面的刷新，仅仅是改变 url。
  • 传递的参数可以放在url里，这样下一个页面就可以通过search/params的值得到了，缺陷就是是信息会直接显示在url上，没有考虑安全问题
  • 在原生的window对象中只有navigator、location、document、history、screen属性；this.props.match实际上属性react路由中的部分。

shell

  • ./ 当前目录，../ 父级目录
  • History指令可以查看shell中使用的命令
    • ！num可以重新执行第num条命令
    • ！！可以重新执行最后一条命令
