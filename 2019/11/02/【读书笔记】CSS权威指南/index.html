Cascading Style Sheets

CSS和文档

发展历史：从关注页面结构过渡到开始关注页面样式了。

谷歌在给页面权重时会考虑到页面的结构化程度。

所谓的层叠（cascading指的就是发生冲突时遵循的规则）。

读者样式表，用于帮助色盲之类的用户定制自己的页面。

添加CSS后可以减少下载时间，替代HTML中font元素，删除额外的标记和结构。

每个元素都生成一个盒，并不是没有元素都用同样的方式创建，依据创建形式分为了替换元素和非替换元素。

替换元素：表示用来替换元素内容的部分并非由文档内容直接表示，如img、input

非替换元素：大多都是非替换元素，直接由文档内容表示

块元素和行内元素，行内元素可以继承块元素但是反过来不行

@import与link的差别在于命令的语法和位置

选择器

CSS的规则为：选择器{声明块}，所以才有各种元素选择器、id选择器、类选择器、选择器分组之类的。

声明中使用了不正确的属性或值则整个声明都会被忽略，CSS样式没有被应用很有可能是属性或值不正确。

针对ID选择器的设置来说，浏览器其实并不会检查HTML中ID的唯一性。但是对编写DOM脚本可能又挑战，getElementById()

伪类（已访问、未访问），伪元素选择器（在文档中插入假想的元素，首字母、首行、之前、之后元素）可以为文档中不一定具体存在的结构指定样式。

结构和层叠

结构树是CSS继承的核心。

元素应用哪些值时要考虑继承、声明的特殊性以及声明的来源，这个过程就叫层叠。

层叠规则：“0000”（特殊性）（第一个样式是为内联样式保留的）（特殊性从3个值改变为4个值时CSS2到CSS2.1的改变）

写在一起的声明是会被解组的，所有可能存在一个针对一个标签的特性，前一半有效而后一半无效。

！important一定要放在声明的最后，位置错误则整个声明都无效

注意有些属性不能被继承，如边框。继承的值没有特殊性，没有特殊性比0特殊性的重要性还要低。（0特殊性：通配符）

（按来源排序）创作人员的样式要胜过读者样式；有！important标志的读者样式要胜于其他所有样式；包括有！important标志的创作人员样式；创作人员和读者样式均比用户代理强

按来源->按特殊性->按顺序（与元素匹配的最后一个选择器胜出）

值和单位

颜色

长度

  • 绝对长度单位（in/cm/mm/pt/pc）（后两个，点和派卡都是印刷单位）
  • 相对长度单位（em/ex）
    • em表示一种给定字体的font-size（可以随字体大小缩放）
    • ex表示的是字体中小写x的高度
    • 像素长度（因为其他设备的原因被看作相对长度，其实在web显示中更像是绝对长度）

URL

​ url和开始括号之间不能有空格，存在空格则整个声明都无效

字体

样式表设置一个字体样式，用户代理根据对应的字体在字体库里寻找一个展示出来。如果找不到就只能用默认字体进行显示（与word相同）

在字体粗度上，只要一个关键字对应的变形不会比前一个关键字对应的变形更细都是允许的（因此，100、200、300都可能映射到同样的粗细变形）

字体加粗时的关键词设定会考虑父元素的字体大小在父元素的基础上进行加粗。

字体大小，font-size的作用就是为给定字体的em框提供一个大小，但实际显示的字符不一定占据全部大小

  • 绝对大小（绝对大小关键字，xx-small、x-small、small、medium、large、x-large、xx-large，两关键字之间的缩放因子一般为1.5）（一般来说medium表示用户代理默认字体大小，在explorer中默认大小时small）
  • 相对大小（关键字只有smaller和larger，使元素的大小相对于其父元素大小上移或者下移）
  • 百分数（根据从父元素继承的大小来计算，实际上Web浏览器很有可能将这个值取整为与之相近的整数像素）（在字体继承中继承的是计算值而不是百分数）
  • 也可以使用长度单位px

字体的风格（font-style、font-variant字体变形[small-caps小型大写字母、normal、inherit]、font-stretch缩放、font-size-adjust当所用字体不是首选字体时的调整值）

font：font-style||font-variant||font-weight||font-size||/line-height？||font-family

  • 前3个属性的位置可以交换
  • font-size和font-family的顺序不可变，且必须有，否则规则无效可能被用户代理忽略
  • 在字体中也可以设置line-height，可选属性，用斜线/分割
  • 忽略值会重置为默认值

文本属性

text-indent（num/precent）块级元素的首行缩进，可继承

text-align（left/center/right/justify/inherit）文本水平对齐属性

line-height

  • 文本行基线之间的距离，inline-box行内框，行框；
  • 如果指定为normal，用户代理一般设置为字体大小的1.2倍；百分数等值都是相对于元素的font-size进行设置；
  • 当一个块级元素从另一个元素继承line-height时要从父元素的font-size上计算而不是子元素的font-size，line-height:1em；
  • 对比在指定一个数的时候，将其作为缩放因子，块内的子元素将继承缩放因子的值而不是后续的计算值，各个元素根据自己的font-size计算对应的line-height

vertical-align（baseline/text-top/top/middle/bottom/text-bottom/sub(相对于父元素的基线降低，距离取决于用户代理)/super/百分数(升高/降低基线)/数值(基线位置更改)），只用于行内元素和表单元格，不可继承。

word-spacing 字间隔，可被text-align的属性更改，会被子元素继承

letter-spacing 字母间隔（优先级以此为text-align=>word-spacing=>letter-spacing），会被子元素继承

text-transform（uppercse、lowercase、capitalize[只对每个单词的首字母大写]、none、inherit）

text-decoration（underline/overline/line-through/blink）

text-shadow（CSS2.1去掉了）

white-space（normal[丢掉多余空白符，换行转空格]/nowrap[防止元素中文本换行，除非br]/pre[保留空白符、保留换行符、不允许自动换行]/pre-wrap[保留空白符、保留换行符、允许自动换行]/pre-line[合并空白符、保留换行符、允许自动换行]）

基本视觉格式化

块级元素

如果没有设置颜色，边框(border)将取元素内容的前景色

行内元素的摆放方式并不直接依赖于包含块

水平属性中只有width、marginLeft和marginRight可以设置为auto，如果设置为auto，设置为auto的属性会自适应宽度使宽度等于父元素的width。如果长度不符合的话，按语言顺序重置marginRight/marginLeft为auto。

如果三个属性都设置为0，则两个外边距都会设置为0，width会尽可能的宽。

水平外边距并不会合并。

通过指定外边距为负值得到更宽的子元素，因为在计算时要求7个水平参数的和为父元素的width，因此该元素自身的width会计算得到一个偏大值（width设置为auto时），当左外边距为负值时就很有可能出现字体超出文本边框的情况。（只有外边距能为负值）

对于替换元素而言，如果width为auto，则元素的宽度为内容的固有宽度。如果一个替换的元素的width不同于其固有高度，那height也会成比例变换，除非显式设置为一个特定值。

如果元素内容高度大于元素框的高度，则由overflow来定义用户代理的行为。

只有height、marginTop和marginBottom可以设置为auto，下边框、边框默认值为0。需要注意的是，当marginTop和marginBottom都被设置为auto时，他们会被自动计算为0，因此并不会用设置auto的方式去实现竖直居中。

如果没有显式的指出包含块的height，那么百分数也会被重置为auto，因为百分数的基数是依据父元素的height，即父元素content区域的height。

如果将块级正常流的高度设置为auto，其默认高度为子元素最高外边框到最低外边框的距离（与行框行内框的关系比较相近），只考虑外边框而不考虑外边距，可能导致子元素的外边距超过父元素。不过如果块元素有上下内边距或者上下边框的话（padding/border）高度为子元素外边距高度差。感觉还是要考虑父元素content area是否直接和外界相连。

相连的外边距会沿竖轴合并/重叠，只会发生在只有margin的情况下，如果有内边距/边框就不会合并。（对负外边距同样有效）

列表项的圆点/数字什么的并不是列表项内容区的一部分，list-style-position将决定是否讲标志项和内容区放到一个块级元素中。

行内元素

匿名文本：未包含在行内元素中的字符串

非替换元素的内边距、边框、外边距等对行内元素或其生成的框没有垂直效果，不会影响元素行内框的高度，所以给span设置margin根本没有用。

会应用上来但不会造成任何影响

在块级元素上声明line-height将为该块级元素的内容设置一个最小行框高度

行内框根据vertical-align的属性垂直对齐（可以是数值，表示基线移动的距离）

增加特别元素的line-height以免出现边框覆盖（可以通过将line-height的值设置为font-size的倍数）

替换元素的整体（padding、margin）被看作是行内框，由于行内框变化从而影响了行框的高度，但是line-height不会被影响，line-height存在的意义是为了在 垂直对齐时使用，对图像等替换元素的行内框没有任何影响。（对比非替换元素边框不会影响行框）

默认的行内替换元素位于基线上，替换元素没有自己的基线，因此下外边距边界与基线对齐。

行内元素可能是块元素的后代，但块元素不可能是行内元素的后代。

行内块元素会作为替换元素放在行内，即底部再文本行的基线上；如果元素框的width被声明为auto，则元素框会自动收缩以适应内容。

css2的display有一个属性为run-in，如果一个元素生成run-in框，而框后面是一个块级框，那么该run-in元素将会成为块级框开始位置的行内框，否则run-in框本身就是块级框。

浮动和定位

浮动

所有元素均可浮动（）

浮动元素的外边距不会合并

如果要浮动一个非替换元素必须要为该元素声明一个width

Float:none用于防止元素浮动，none也是float的默认值

包含块（containning block）是浮动元素最近的块级祖先元素（所以浮动元素要显式声明width）

利用负边距向上、向左浮动

CSS1，2明确指出用户代理不必重新显示已显示的内容来适应文档中后出现的内容

行内框与浮动元素重叠时，all都在浮动元素之上；块框与浮动元素重叠时，边框和背景在浮动元素之下，内容在浮动元素之上。

clear的原理是在元素的外边距之上在增加额外的间隔，不允许任何浮动元素进入该区域，元素的外边距并不会改变。

定位

基于正常位置，基于父元素，基于浏览器窗口（都是相对包含块定位）

（position：static/relative/absolute/fixed/inherit）

  • static ，正常生成，文档流的一部分
  • relative，元素框偏移某一距离，元素仍保持未定位前的形状，原本的空间被仍被保留
  • absolute，相对包含块（实际应该是定位上下文）定位，文档流中的位置被删除，重新生成块级框（包含块为最近的position值不是static的祖先元素）
  • fixed，包含块是视图本身，其他同absolute

元素当然可能在包含块之外，设置为负值即可。

除了长度和百分数，偏移属性还可以设置为auto，默认值。需要注意的是当指定top、bottom、left、right等时同时设置了width和height，此时width和height并没有意义，但是当两者冲突时width和height的优先级更高一些。需要注意的是width和height指定的是内容区的范围，如果还设置了padding、border等，定位元素可能超过包含块。

如果有需要可以针对内容框设置最大最小范围，min-width、min-height、max-width、max-height

内容的溢出

overflow：visible（元素框外也可见）、hidden（被剪裁无滚动条）->clip(设置剪裁区域将剪裁区域延伸到元素框之外，这会扩展看到内容的区域但不会改变内容流，历史悠久浏览器实现方式不一致)、scroll（滚动条显示）、auto、inherit

元素可见性

visibility：visible、hidden（对比display:none和transparent）、collapse（如果用于非表元素，与hidden的含义相同）、inherit

visibility属性可以继承，一个hidden元素的后代可以设置为visible

绝对定位（absolute）

元素绝对定位时，会从文档流中完全删除，定位元素不会流入其他元素因此可能会出现覆盖的情况

绝对定位元素的包含块时最近的position值不为static的祖先元素，一般来说会选择一个元素作为绝对定位元素的包含块，并将其position设置为relative而且没有偏移。

由于元素依据文档流定位所有定位元素可以随着文档的滚动而滚动。

对于绝对元素来说他的大小与摆放的位置有关（当height和width都为指定时），当将top和left设置为auto。（top设置为auto会和元素未定位前的本来顶端位置对齐，left和right同理，但是bottom取不到静态位置）

当height、width的高度和位置设置冲突时会考虑优先忽略right的值以满足left和width，如果同时指定了auto会优先利用auto的属性进行填充满足。

对于垂直来说与水平相似优先忽略bottom，利用计算值覆盖，但是margin设置为auto可以得到垂直居中的效果。

但是对于替换元素而言，其他相似，如果bottom的值为auto（left、right依据阅读顺序看要不要将auto更改为静态位置），则将margin-top、margin-bottom的auto值设置为0

Z-轴决定的相同位置的两个元素的叠放顺序，数值越大的离读者越近。但是子元素是依据祖先元素设置自己z-index的。对于auto来说可以将其处理为0，而负值则会在body后如果body不透明就不可见了。

固定定位（fixed）

包含块时视窗，元素会完全从文档流中去除，例如导航栏，锚点。

相对定位（relative）

原空间位置不会消失，新建的包含块对应元素原本的位置，如果top、bottom、left、right矛盾会优先考虑top、left。

表布局

css的内部表元素有内容、边框和内边距但是没有外边距，通过margin定义单元间隔都是会被忽略的。

为了区分table元素，定义了display的一些属性，table生成块框的矩形块、inline-table生成行内框的矩形块、table-row指定一个元素是单元格的行……

![image-20191011222143099](/Users/huxiaohui/Library/Application Support/typora-user-images/image-20191011222143099.png)

css的列和行只能接受border、background、width（定义了最小宽度）、visibility的定义，如果visibility的值不是collapse则会被忽略，collapse表示该列或列祖的所有单元格都不显示。

可以通过设置display的值将遗漏组件作为匿名对象插入table

table的层次顺序由近到低：单元格、行、行组、列、列组、表。默认背景都是透明的。

border-collapse：collapse、separate，决定了两个单元格的边框是否合并，因为每个单元格都会有一个边框。当border-collapse的值为collapse时会出现单元格合并的问题，这时候用哪个边框主要是看border-style，hidden的优先级最高、none的优先级最低，窄边框的优先级不敌宽边框；当宽度也相同则考虑边框样式，double>solid>dashed>dotted>ridge>outset>groove>inset

表大小

table-layout：auto、fixed（固定布局、自动布局）显著差别在于速度，使用固定宽度的表布局时，用户代理的计算代理会更快。

固定布局不依赖于表单元格的内容，所有列的宽度都由表的第一行定义，未指定宽度的列会平均分剩余的width，其他行定义的width会被忽视。

一般来说width为auto就会触发自动布局，不管table-layout的值。这时候的表布局取决于单元格的内容，用户代理在查看完表的所有内容之前无法确定表的布局。

如果一个列的width为百分数，则这个百分数要相对于表的宽度计算，因此要先将百分数保存下来，等得到了表格剩余部分的宽度后再进行计算。

用户代理针对高度可能有不同的做法所有一般来说要避免设置高度

列表与生成内容

列表

列表的项其实是一个块框，多出来的部分不属于文档布局，由用户代理格式化。

css无法区分有序列表和无序列表，默认是实心圆做标志，但是用户代理可能会进行一定的设置。如果list-style-type设置为none则表示不显示任何样式，属性可继承。

利用list-style-image将标志设置为自定义的图片；

list-style-position确定标志出现在列表内容外还是列表内容内；

如果标志放在列表项内容之外，他们不会影响其他元素的布局也不会影响列表项本身的布局，就是离内容边界有一段距离，只要内容边界移动，标志也会跟着移动。标志总是相对于列表项的内容放置。

css3中引入了::marker进行相关的设置（实验ing）

生成内容

指由浏览器创建的内容，如列表标记。文档中插入生成内容可以利用:before和:after伪元素，利用content属性添加内容。除列表标志外，其他生成内容都在元素框内部。

css2中如果:before或:after的主体是块级元素，则display只接受none、inline、block、mark，其他值为block；同理，如果是行内元素，则只接受none和inline，其他值为inline。

除了文字还能设置计数器，利用counter-reset可以设置计数器起点，counter-increment用来设置递增间隔。需要注意的是重置发生在显示前，因此显示的一定是默认值+increment（word中的标题编号）
