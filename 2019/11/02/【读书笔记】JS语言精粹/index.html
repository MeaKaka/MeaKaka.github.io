【读书笔记】JS语言精粹

Posted on 2019-11-02 | Post modified: 2020-03-09
Words count in article: | Reading time ≈

概述

JS没有块作用域的概念，所有编译单元的顶级变量都会被撮合到全局对象的公共命名空间——>(这里体现的闭包的意义)

因为缺少链接器所以JS将他们抛到了全局空间

代码块不会创建新作用域

var语句在函数内部时定义的是函数的私有变量

JS的核心是指的操作浏览器API和文档服务对象

语法

JS没有分离出整数类型（1和1.0的值相同），内部只有64位浮点数一种数据类型

NaN不等于任何值包括他自己

JS中有对象Math用来做数字处理

JS被创造时，unicode是16位的字符集，所以JS中的所有字符都是16位的

字符串是不可变的，一旦字符串被创造就永远无法改变它

两个包含完全相同字符且字符顺序也相同的字符串被认为是相同的字符串

对象

object也可以用for in枚举对象的所有键名，但是for in并不保证顺序而且有可能有一些是不需要的，是存在在原型链上的，可以利用typeof（排除函数）、hasOwnProperty方法进行过滤

JS里的对象是无类型的，正确的使用原型链可以减少对象初始化时消耗的时间和内存

对象的属性名可以是包含字符串在内的任何字符串，一定是字符串单不一定要用引号括住

引用不一样的空对象

删除对象的属性可能会让原型链中的属性透现出来

函数

JS中函数就是对象（Function.prototype->object.prototype）

除了声明时定义的形式参数，每个函数还接收两个附加参数（this，arguments），当时间参数和形式参数的个数不匹配时不会导致运行错误，超出的参数会被忽略、不足的参数会被定义成undefined

arguments拥有一个length属性，但它没有任何数组方法

函数总是会返回一个值，如果没有指定返回值就会返回undefined，如果函数调用时在前面加上new前缀，且返回值不是一个对象，则返回this为新对象

c有尾递归、python、js等均无尾递归所谓尾递归就是 递归语句在在末尾，一般来说在递归过程中如果要保存每个递归流程中的状态或是变量值就是占据很大的内存空间，而尾递归则通过将变量传递给一个简单递归函数，实现堆栈的常量更新，而不是每一次都申请新的空间。每次调用都在收集结果

利用存储创建有记忆的函数

继承

不是对象到对象的继承，而是有一个中间层——构造器函数

没有私有变量和私有函数，对象的所有属性都是可见的=>应用模块模式，通过var在constructor定义私有实例变量， 函数化，用函数去操作变量

数组

没有数组，只有类数组，实质是对象，将数组下标转化为属性。

但是即使是表面看起来相同的数组和对象，其内在也有所不同，因为他们继承的来源不同，array.prototype会涵盖很多与数组有关的方法，也有一个length属性。

为什么说JS数组内元素类型可以各种各样是因为本来就不是一段线性空间，所以不涉及申请空间的问题。

数组的length属性是数组最大整数属性+1，不一定等于属性的数量（比如删掉其中的某个属性），length也不会出现越界问题，arr[length+8]，则会在创建一个新的元素（实质就是object，add一个新的元素）；length设小会导致超出部分被删除

正则表达式

RegExp对象有三个标识，g(全局)、i(大小写不敏感)、m(多行)

方法

JS提供了一套可用在标准类型上的标准方法集

Array相关的方法集：

array.concat，新数组，浅复制

array.join， 原来大量字符串片段组装成一个字符串，利用数组的join比利用+元素更快，但是后来+有做了一些优化，性能明显高于join，大多数情况下还是要首选+运算符

array.pop/array.push，返回处理元素

array.reverse，返回更新后的数组

Array.shift，返回元素，比pop要慢，因为针对后面的元素还有移动时间

array.unshift，插入到开始位置，返回array的新的length

array.slice(start,end) ，浅复制新数组

Array.splice(start, deleteCount, item…)，移除一个或多个元素并用新的item代替他，返回包含被移除元素的数组

array.sort，排序，字符串顺序排序，可以指定自己的sort函数array.sort(function(a,b){…return…})

Number有关的方法集

number.toExponential(num) 将number转换为一个指数形式的字符串，num表示小数点的位数值为0～20

number.toFixed(num) 将number转换为一个十进制字符串，num表示小数点的位数值为0～20

number.toPrecision(num) 将number转换为一个十进制字符串，num表示字符串的精度，数值为0～21

number.toString(radix) 将number转换为字符串，radix表示基础2～36，默认是10

RegExp有关的方法集

regexp.exec(string) 捕获的文本数组

regexp.test(string) true/false

String有关的方法集

string.charAt(pos) 以字符串的形式返回pos位置的字符

string.charCodeAt(pos) 返回pos位置字符的字符码位

string.concat() 没有+好使

string.indexOf(string,pos) 从pos位置开始寻找，返回位置或者-1

string.lastIndexOf(string,pos) 从pos位置开始寻找，返回位置或者-1,从后向前寻找

string.localeCompare(func) 比较两个字符串

string.match(regexp) 对比regexp.exec(string)

string.replace(searchValue，replaceValue) 返回新的字符串，无g仅匹配一次，replaceValue可以是个函数，用函数返回值做替换文本

string.search(regexp) 忽略g，返回第一个匹配的首字符的位置

string.slice() 新字符串

string.split(separator，limit) 字符串数组，limit表示被分割的片段数量

string.substring(start,end) 与slice相同，不能处理负参数，一般来说使用slice就够了

string.toLocaleLowerCase() /string.toLocaleUpperCase()返回新字符串，主要用于土耳其语

string.toLowerCase() /string.toUpperCase()返回新字符串

string.fromCharCode(code, code,….) 根据数字编码返回一个字符串

代码风格

null

优美的特性

函数是顶级对象

基于原型继承的动态对象

对象字面量(方便的按指定规格创建新对象的表示法，key/value对)和数组字面量

Awful Parts

全局变量

可以被程序的任何部分在任何时间修改，而且和子程序变量重名导致一些问题

定义全局变量：函数外放置var、给全局变量如window添加属性、直接使用未声明的变量（隐式全局变量）

作用域

有类C的块语法却没有提供提供块作用域，

代码块中声明的变量在包含此代码块的函数的任何位置都是可见的

自动插入分号

可能导致程序被误解

保留字

不能用来命名变量和参数，点表示法中强制加“”

Unicode

JS的字符是16位的，

而unicode通过不停的填充和分区定义有2^5*2^16个字符。最开始的65536个字符为所在平面被称为基础平面，16进制表示为U+0000——U+FFFF；后续的字符被放在了辅助平面（U+010000——U+10FFFF）。unicode规定的就是字符的码点，至于编码方式什么的就是utf-8（可变长）、utf-32（unicode的值，浪费）、utf-16（基础平面2个字节、辅助平面4个字节）…..

在unicode的基础平面里U+D800到U+DFFF是空段，正好可以做映射，辅助平面的字符被拆成了两个字符

JS（1995）用的是unicode字符集但是用的编码方式是UCS-2（1990），对比utf-16（1996）。因此JS在读取字符时要进行额外的处理，判断是否需要连着后续的两个字节都接着读一下。

ES6增强了unicode的支持，可以自动识别四字节码点，允许用码点表示unicode字符

typeof

typeof（null）会返回一个object

parseInt

将字符串转换为整数，但是遇到非数字时就会立刻停止解析

如果字符串第一个字符是0就基于八进制而不是10进制来求值，不过可以指定转换的参数

+可以针对字符串和数字进行操作

浮点数

浮点数精度

NaN
伪数组
假值

0、NaN、’ ‘、false、null、undefined

hasOwnProperty

是一个方法，在for in时用来避雷，但是他们可能被更改或替换而产生雷区

Bad Parts

==与！=

强制类型转换

with

本意是提供一个访问深度嵌套对象成员的快捷方式，但是新建成员时XXXXXXXXXXX。永远不要使用with，用var代替他

eval

对字符串做计算，功能强大，危险

json.parse()

continue

性能问题

switch

case语句后要有break，明确中断流程，否则容易导致意外的穿越

++ –
位运算符

位运算符与Java相同，但是Java处理的是整数，JS只有双精度的浮点数，因此要先转化为整数、执行计算再转换回去。很慢

function语句和function表达式

用var foo = function foo(){}

而不是function foo(){}

因为function解析时会被提升

类型的包装对象

new Boolean、new Number、new Object….没必要

new

函数还是构造函数，没有用new，this就是绑定到全局变量，在不知情的情况下修改会对全局变量造成污染

void

JS中的void是一个运算符，接收运算数，返回undefined，不知道有什么用
