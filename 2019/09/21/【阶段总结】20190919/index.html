跨域相关

跨域的解决方法（前端+后端）（nginx，jsonp，cors）

cors

  • 简单请求
  • 预请求

在跨域请求中，若服务端返回了正确的跨域响应部首， 则跨域请求能正常获取数据，即response中需要有一定的字段：Access-Control-Allow-Origin、Access-Control-Allow-Method、Access-Control-Allow-Headers、Access-Control-Allow-Credentials（是否提供证书，携带cookie传递）（omit表示从不发送cookies；same-origin表示只有当URL和响应脚本同源时才发送cookie；include表示不论什么情况总是发布）（以上均为http请求中的字段内容）

对比XHR中的withCredentials属性（XHR对象是一组发送请求解析响应的API）

当https中不能将access-control-allow-origin设为通配符，设置为\则默认不需要提供cookie。

access-control-allow-origin*与Access-control-allow-credentials：true冲突。

Web项目在本地运行时node会在本地创建一个假服务器，通过将localhost的请求更改为sandbox请求，接收后端的response然后再转发给前端，在浏览器的调试窗口不可见node的转发部分。

与后端的数据交互

JS的boolean为false、true；python的boolean为False、True，有时候可以约定用数字传递。

利用JSON传递时可以传递数据的类型，但是当传递的数据格式是FormData时，所有的数据都变成了字符串。

当FormData作为数据格式的时候，post请求会将Data序列化，按照key1=val&key=val2的方式进行编码，key和val都会被进行URl转码。

后端是如何区分前端传递的数据的格式的呢？通过content-type字段的值进行判断从而进行解码。常见的content-type字段的类型

  • application/x-www-form-urlencoded（浏览器原生form表单，按formData样式解码）
  • multipart/form-data
  • application/json（消息主体是序列化后的 JSON 字符串，时候Restful API接口）
  • text/xml（使用 HTTP 作为传输协议，XML 作为编码方式的远程调用规范）

事件的捕获和冒泡

有的时候会发生postman模拟请求没有问题，但浏览器的请求一直在请求状态的情况，很有可能是页面自动触发到了其他请求，然后又切回了页面请求，一直处在请求的循环中。

使用event.stopPropagation()起到阻止捕获和冒泡阶段中当前事件的进一步传播。如果当你点击click的时候，只想触发绑定在click上的监听函数，而不想触发“传播链”上的其他函数，那么则使用stopPropagation，如触发<a>中的默认链接行为（刷新页面）。

最常见的情况的是antd的popconfirm组件，在onconfirm(本质是一个click事件)回调时要注意阻止事件的传播。

此外，在页面的回退时，为了保存当前的页面的状态会用到this.props.history.replace方法，但是后续如果没有对应的页面跳转，如<a>，则会针对当前页面一直渲染。

对比event.preventDefault()表示取消默认事件，它是事件对象(Event)的一个方法，作用是取消一个目标元素的默认行为。既然是说默认行为，当然是元素必须有默认行为才能被取消，如果元素本身就没有默认行为，调用当然就无效了，如<a>。

event代表事件的状态，例如触发event对象的元素、鼠标的位置及状态、按下的键等等；event对象只在事件发生的过程中才有效。

直接return false可以实现preventDefault和stopPropagation的功能

React

react中父子组件的执行顺序：

  1. father constructor
  2. father getDerivedStateFromProps
  3. father render
  4. children constructor
  5. children getDerivedStateFromProps
  6. children render
  7. children componentDidMount
  8. father componentDidMount

React出现 “不要给unmount的组件赋值state” 的情况，可能是由于组件在加载时发送了异步请求然后组件切换了，切换后接收到了后端的返回值但是请求/展现返回值的组件已经被卸载了。

componentWillReceiveProps props更新时component被重新渲染（React v16已废弃）

export default XXXXX 表示在引用该组件时，不管是什么名字都对应此组件

state 的更新过程是同步执行的，也就是说，action 更改 state 后，新的 state 可以立即被获取。

JS

Object.assign()方法用于将所有可枚举属性的值从一个或多个源对象复制到目标对象，如果源对象的属性和目标对象的属性有重复时，因为复制的缘故所有以源对象属性值为准，它将返回目标对象。（x2）

json.parse()将json字符串转换为对象

字符串和数字拼接出现多个object的情况，可能是因为用了嵌套的{}

append针对DOM节点，被选元素；pop、push针对数组

date.setMonth(date.getMonth() + 1)，不要认为 month + 1 就是月份加 1，它的意思是当前日期加上当月天数 。因此很有可能在+1时出现跨月的情况。想获取上个月最后一天，只需要设置SetDate参数为0即可。

window.postMessage() 方法可以安全地实现跨源通信。（通信的两个页面可以跨源）

token可以通过headers.get的方式获取。后端通过用户id、密码、时间之类的生成的一串序列号，由用户（浏览器）保存在session storage中，当发送的请求需要token字段时，再从浏览器中的sessionStorage里利用getItem获取。sessionStorage.getItem(“token”)

Tooltip的样式的更改可以利用overlaystyle，宽度通过设置minwidth来修改。

getFieldDecorator装饰组件相当于对数据做了渲染，直接以某一个组件的形式呈现处理，是antd组件form里的属性。
